# Failure Analysis

Use this file to record observed policy failures from evaluation and video review.

## Failure Categories

- Early fall.
- Repeated stumbling.
- Poor command tracking.
- Terrain-specific instability.
- Push recovery failure.
- Foot scuffing or dragging.
- Excessive body pitch or roll.
- Policy collapse under randomized dynamics.

## Entry Template

```text
Run:
Checkpoint:
Task:
Seed:
Evaluation seed:
Condition:
Observed behavior:
Likely cause:
Evidence file:
Next action:
```

## Reporting Rule

Failure cases should be represented in the final README when they materially affect the robustness claim. Do not present only the best rollout if sweep data shows consistent weaknesses.
